Welcher technische Ansatz für die Homepage?

Alle Optionen nutzen den bewährten Stack aus deinem Workspace — die Frage ist: wie viel Framework brauchst du?

A

Vite + React + React Router (Empfohlen)

Der gleiche Stack wie alle deine Web-Projekte. SPA mit Client-Side Routing. Tailwind CSS + shadcn/ui. i18n mit react-i18next.

Pros

  • 100% konsistent mit deinem Workspace (ATS, Costtracker etc.)
  • Schnellstes Setup — du kennst den Stack
  • Einfaches Vercel-Deployment
  • shadcn/ui Komponenten wiederverwendbar

Cons

  • Kein SSR → SEO nur mit Prerendering (vite-plugin-ssr oder prerender)
  • Legal-Seiten nicht direkt crawlbar ohne Extra-Arbeit
B

Next.js (App Router)

React-basiert aber mit Server-Side Rendering. Perfekt für SEO-relevante Seiten. i18n mit next-intl. Vercel-native.

Pros

  • SEO out-of-the-box (SSR/SSG)
  • Legal-Seiten als statische Pages → perfekt crawlbar
  • Vercel-native — Zero-Config Deployment
  • i18n mit URL-Prefix (/de, /en) eingebaut

Cons

  • Neuer Stack im Workspace (kein bestehendes Next.js-Projekt)
  • Etwas mehr Komplexität (App Router, Server Components)
C

Astro + React-Inseln

Static-first Framework. HTML wird zur Buildzeit generiert. React nur wo Interaktivität nötig ist. Extrem schnell.

Pros

  • Beste Performance (nahezu Zero JS shipped)
  • SEO perfekt (rein statisch)
  • Content Collections ideal für Legal-Seiten (Markdown)
  • i18n mit Astro-Bordmitteln

Cons

  • Komplett neues Framework — anderer Workflow
  • shadcn/ui nicht direkt kompatibel
  • Weniger Ökosystem-Erfahrung in deinem Workspace
